# Process Monitor

## Purpose

Processes represent active activity on a system.

Understanding process behavior is essential for
security reviews and operational awareness.

---

## Why Processes Matter

Every application becomes one or more processes.

Monitoring them provides visibility into:

- Resource usage
- User activity
- Network activity
- Service behavior

---

## Areas of Interest

### CPU Usage

Unexpected spikes may indicate:

- Misconfiguration
- Resource exhaustion
- Unauthorized activity

### Memory Usage

High memory usage can affect stability.

### Network Activity

Processes communicating externally should be understood and expected.

### Privileged Execution

Processes running with elevated privileges deserve additional scrutiny.

---

## Operational Review

Questions:

What is running?

Why is it running?

Who started it?

Does it require elevated privileges?

Does it communicate externally?

---

## Key Principle

Visibility comes before trust.
